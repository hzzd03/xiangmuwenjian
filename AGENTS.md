# AI 开发助手使用说明

本文档为后续开发 AI 提供项目结构说明和开发规范。

## 文件夹结构说明

### 📁 docs/ - 产品文档区
**用途**：存放所有产品相关文档
- PRD（产品需求文档）
- 需求迭代记录
- 关键技术决策文档
- 项目计划和时间线

**规则**：
- 所有产品级文档必须放在此目录
- 文档命名格式：`[日期]-[文档类型]-[简短描述].md`
- 示例：`2024-01-15-PRD-用户登录功能.md`

### 📁 assets/ - 资源文件区
#### assets/design/ - 设计素材
**用途**：存放设计相关文件
- UI 效果图
- 设计稿
- UI 参考图

**规则**：
- 图片格式建议：PNG、JPG、SVG
- 命名格式：`[功能模块]-[类型]-[版本].png`
- 示例：`login-ui-v1.0.png`

#### assets/bug/ - 测试报错截图
**用途**：存放测试和调试过程中的截图
- Bug 截图
- 错误信息截图
- 测试用例截图

**规则**：
- 命名格式：`[日期]-[bug描述].png`
- 示例：`2024-01-15-登录失败弹窗.png`

#### assets/reference/ - 参考图和灵感收集
**用途**：存放参考素材和设计灵感
- 竞品截图
- 设计灵感图
- 参考案例

**规则**：
- 按类别分子文件夹
- 标注来源和用途

### 📁 notes/ - 学习笔记区
**用途**：记录开发过程中的学习和问题解决
- 踩坑记录
- 技术方案笔记
- 学习心得

**规则**：
- 一个主题一个文件
- 命名格式：`[技术主题]-[日期].md`
- 示例：`React性能优化-2024-01-15.md`

### 📁 code/ - 代码区
**用途**：存放所有项目代码
- 源代码
- 配置文件
- 构建脚本

**重要规则**：
⚠️ **代码区不引用外部文件**：所有依赖和资源必须在 code 目录内
⚠️ 相对路径引用：使用相对于 code 目录的路径
⚠️ 资源引用：如需引用图片等资源，统一从 assets 目录引用

## 开发规范

### 1. 文档管理规范
- 所有文档统一放在 `docs/` 目录
- 使用 Markdown 格式编写文档
- 文档必须包含标题、日期、作者信息

### 2. 代码管理规范
- 代码必须放在 `code/` 目录内
- 不允许在代码目录外创建代码文件
- 使用清晰的目录结构组织代码

### 3. 资源管理规范
- 设计时资源 → `assets/design/`
- Bug 截图 → `assets/bug/`
- 参考素材 → `assets/reference/`
- 代码需要的静态资源 → `code/assets/`（在 code 内部创建）

### 4. 命名规范
- 文件名：使用中划线分隔，小写字母
- 文档名：可包含中文，格式清晰
- 代码：遵循所用语言的命名约定

### 5. 版本控制规范
- 频繁提交，每次提交解决一个问题
- 提交信息清晰明确
- 提交前检查是否包含不必要的文件

## AI 助手工作流程

### 开始任务前
1. 阅读本说明文档
2. 查看 `docs/` 目录了解项目需求
3. 查看 `notes/` 目录了解已知问题和解决方案

### 开发过程中
1. 代码文件放在 `code/` 目录
2. 文档更新放在 `docs/` 目录
3. 学习笔记记录在 `notes/` 目录
4. 截图资源放在对应的 `assets/` 子目录

### 任务完成后
1. 更新相关文档
2. 记录重要决策和踩坑经验
3. 清理临时文件
